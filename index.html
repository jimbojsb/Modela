What is Modela?

Modela is an abject-oriented CouchDB client for PHP. It provides both basic connection and querying features and base objects that can be used for building domain models.

What features does Modela Have?

  • Light-weight HTTP layer
  • Extensible document base class for creating models
  • Autoloading of classes that won't conflict with other libraries
  • Command line tools for managing view definitions
  • Zend Framework plugins for bootstrapping and authentication

Where and how do I get it?

Modela is hosted and maintained on GitHub. You can download the latest source version directly from GitHub if you like. The preferred method for getting Modela is to install via PEAR. You can install Modela as follows: 

# pear channel-discover mode.la
# pear install modela/Modela-0.2.2

Usage

Modela is easy to integrate into your project. The first step is getting a connection set up. Modela has a "Core" (Modela_Core) object that manages connection information and handles the dispatching of requests. As CouchDB uses HTTP as it's transport protocol, Modela_Core doesn't actually need to maintain any connections. It simply holds the information for use when making individual HTTP requests.

Configure Modela_Core

<?php
    $modela = Modela_Core::getInstance();
    $modela->setHostname('localhost');
    $modela->setDatabase('myapp');
?>

Zend Framework Bootstrap Resource

For Zend Framework applications that are using Zend_Application, Modela inludes a bootstrap resource that allows you to configure Modela from your application.ini. This assumes you installed Modela via PEAR, but as long as Modela is on your include_path, it should work. All you have to do is add the following lines to your application.ini

pluginPaths.Modela_Zend_Application_Resource = "Modela/Zend/Application/Resource"
resources.modela.hostname = [hostname]
resources.modela.database = [databasename]
resources.modela.modelsPath = [path to model classes, optional]
